## 二、各组件详细设计

### 组件一：Crypto 行业动态

#### 2.1.1 功能描述
- 抓取加密货币行业新闻
- 通过 DeepSeek 判断新闻优先级
- 生成热搜榜（综合发布时间、浏览次数、互动量等维度）

#### 2.1.2 Agent 流程设计

```
┌──────────────────────────────────────────────────────────────────────────────┐
│                    Crypto 行业动态 Agent 流程                                   │
└──────────────────────────────────────────────────────────────────────────────┘

[1] 数据采集 Agent
    │
    ├─► 输入: 无（定时触发 / 手动触发）
    ├─► 数据源: CoinDesk, CoinTelegraph, The Block, Decrypt, 金色财经, 链闻等
    ├─► 采集方式: RSS / API / 爬虫（遵守 robots.txt）
    └─► 输出: 原始新闻列表 [{title, url, source, publish_time, content_snippet, ...}]

                    ▼

[2] 数据清洗 & 标准化 Agent
    │
    ├─► 输入: 原始新闻列表
    ├─► 处理: 去重、过滤低质量、统一时间格式、提取关键字段
    └─► 输出: 标准化新闻列表

                    ▼

[3] 元数据补充 Agent（可选）
    │
    ├─► 输入: 标准化新闻列表
    ├─► 处理: 调用各平台 API 获取浏览次数、点赞数、评论数、转发数
    └─► 输出: 带互动数据的新闻列表

                    ▼

[4] DeepSeek 优先级 & 热度评估 Agent
    │
    ├─► 输入: 带元数据的新闻列表
    ├─► Prompt 设计要点:
    │   • 根据新闻主题（监管、技术、市场、安全事件等）判断优先级（高/中/低）
    │   • 根据发布时间、互动数据计算热度分数
    │   • 输出结构化 JSON: {priority, hot_score, summary, key_topics}
    └─► 输出: 带优先级和热度分数的新闻

                    ▼

[5] 热搜榜生成 Agent
    │
    ├─► 输入: 带优先级和热度的新闻
    ├─► 算法: 加权排序 = f(priority_weight, hot_score, recency_decay)
    └─► 输出: 排序后的热搜榜 Top N

                    ▼

[6] 前端展示
    └─► 卡片式列表 / 时间线，支持筛选（按优先级、按来源、按时间）
```

#### 2.1.3 数据流示意

```
新闻源 → 采集 → 清洗 → 元数据补充 → DeepSeek评估 → 热搜排序 → 看板展示
```

### 组件二：公司站内动态（Binance / ByBit / Bitget）

#### 2.2.1 功能描述
- 监控 Binance、ByBit、Bitget 三家交易所 APP 内部的运营活动
- 包括：新币上线、活动公告、产品更新、促销活动等
- 生成站内热搜榜

#### 2.2.2 Agent 流程设计

```
┌──────────────────────────────────────────────────────────────────────────────┐
│                    公司站内动态 Agent 流程                                      │
└──────────────────────────────────────────────────────────────────────────────┘

[1] 站内数据采集 Agent（按公司分 Agent 或统一调度）
    │
    ├─► Binance: 官网公告页、API 公告接口、Launchpad 页面
    ├─► ByBit: 官网公告、活动中心
    ├─► Bitget: 官网公告、新币上线页
    ├─► 采集方式: 官网公告 API / 定时爬取公告列表
    └─► 输出: 各公司站内动态列表 [{company, type, title, url, publish_time, ...}]

                    ▼

[2] 动态分类 Agent
    │
    ├─► 分类: 新币上线 | 活动促销 | 产品更新 | 规则变更 | 其他
    └─► 输出: 带分类标签的动态列表

                    ▼

[3] 热度评估 Agent
    │
    ├─► 维度: 发布时间、活动类型重要性、参与门槛、奖励规模
    ├─► 可选: 若可获取阅读量/参与人数，纳入热度计算
    └─► 输出: 带热度分数的动态

                    ▼

[4] 热搜榜生成 Agent
    │
    ├─► 按公司分别生成热搜榜，或合并展示
    └─► 输出: 站内动态热搜榜

                    ▼

[5] 前端展示
    └─► 三列布局（Binance | ByBit | Bitget）或 Tab 切换
```

#### 2.2.3 数据流示意

```
交易所官网/API → 采集公告 → 分类 → 热度评估 → 热搜排序 → 看板展示
```

#### 2.2.4 站内数据获取方式详解

| 交易所 | 获取方式 | 端点/地址 | 认证 | 说明 |
|--------|----------|-----------|------|------|
| **Binance** | WebSocket 实时推送 | `wss://stream.binance.com:9443/ws` 或 `wss://stream.binance.com:443/ws` | 无需 | 订阅 topic `com_announcement_en`，实时接收英文公告 |
| **Binance** | 官网公告页 | https://www.binance.com/en/support/announcement | 无需 | 备选：定时爬取列表页，有 0–60 秒延迟 |
| **ByBit** | REST API | `GET https://api.bybit.com/v5/announcements/index` | 无需 | 官方 API，支持分页、按类型/标签筛选 |
| **Bitget** | 官网爬取 | https://www.bitget.com/support（公告/新币上线分类） | 无需 | 无官方公告 API，需自建爬虫或 Apify |

**Binance WebSocket 公告流**

- 连接后发送订阅：`{"method":"SUBSCRIBE","params":["com_announcement_en"],"id":1}`
- 返回字段：`catalogId`、`catalogName`（如 Delisting、New Crypto）、`publishDate`、`title`、`body`、`disclaimer`
- 文档：https://developers.binance.com/docs/cms/announcement

**ByBit 公告 API 参数**

| 参数 | 必填 | 说明 |
|------|------|------|
| locale | 是 | 语言，如 `en-US`、`zh-TW` |
| type | 否 | 公告类型：`new_crypto`（新币）、`latest_activities`（活动）等 |
| tag | 否 | 标签筛选 |
| page / limit | 否 | 分页，默认 limit=20 |

- 文档：https://bybit-exchange.github.io/docs/v5/announcement

**Bitget 爬取建议**

- 目标页面：Support 中心公告列表、新币上线专题
- 方案：BeautifulSoup/Playwright 解析 HTML，或使用 Apify 预制爬虫
- 注意：遵守 robots.txt，控制请求频率

### 组件三：公司站外动态

#### 2.3.1 功能描述
- 监控 Binance、ByBit、Bitget 在站外渠道的动态
- 包括：Twitter/X 官方账号、新闻媒体报道、社区讨论等

#### 2.3.2 Agent 流程设计

```
┌──────────────────────────────────────────────────────────────────────────────┐
│                    公司站外动态 Agent 流程                                      │
└──────────────────────────────────────────────────────────────────────────────┘

[1] 站外数据采集 Agent
    │
    ├─► 社交媒体: Twitter/X API (Binance, ByBit, Bitget 官方账号推文)
    ├─► 新闻媒体: 搜索引擎/新闻 API 抓取公司相关报道
    ├─► 可选: Reddit、Telegram 等社区提及
    └─► 输出: 站外动态原始数据 [{source, platform, content, time, engagement, ...}]

                    ▼

[2] 数据清洗 & 去重 Agent
    │
    ├─► 过滤无关内容、合并同一事件的多源报道
    └─► 输出: 清洗后动态列表

                    ▼

[3] 情感 & 重要性分析 Agent（可选 DeepSeek）
    │
    ├─► 分析: 正面/中性/负面、对市场/用户的影响程度
    └─► 输出: 带情感和重要性标签的动态

                    ▼

[4] 热度排序 Agent
    │
    ├─► 综合: 互动量、发布时间、来源权威性
    └─► 输出: 站外动态热搜榜

                    ▼

[5] 前端展示
    └─► 按公司分组，支持按平台筛选（Twitter / 新闻 / 其他）
```

#### 2.3.3 数据流示意

```
Twitter API / 新闻API → 采集 → 清洗去重 → 情感分析(可选) → 热度排序 → 看板展示
```

### 组件四：加密货币价格看板

#### 2.4.1 功能描述
- 展示 Bitcoin、Ethereum 等主流币种的实时价格
- 展示当日、MTD（月初至今）、YTD（年初至今）的价格趋势线

#### 2.4.2 Agent 流程设计

```
┌──────────────────────────────────────────────────────────────────────────────┐
│                    加密货币价格看板 Agent 流程                                  │
└──────────────────────────────────────────────────────────────────────────────┘

[1] 价格数据采集 Agent
    │
    ├─► 数据源: CoinGecko API / Binance API / 其他行情 API
    ├─► 采集内容: 实时价格、历史 K 线（1D 粒度）
    └─► 输出: 价格时间序列数据

                    ▼

[2] 数据聚合 Agent
    │
    ├─► 计算当日开盘价、当前价、涨跌幅
    ├─► 计算 MTD: 本月1日开盘 → 当前
    ├─► 计算 YTD: 本年1月1日开盘 → 当前
    └─► 输出: 结构化价格数据 + 趋势线数据点

                    ▼

[3] 缓存 & 更新策略
    │
    ├─► 实时价格: 每 10–30 秒刷新
    ├─► 日线数据: 每日定时更新
    └─► 使用 Redis/内存缓存减少 API 调用

                    ▼

[4] 前端展示
    └─► 价格卡片 + 三条趋势线图表（当日 / MTD / YTD）
```

#### 2.4.3 数据流示意

```
价格 API → 采集 K 线 → 聚合计算 → 缓存 → 图表渲染
```

### 组件五：聊天机器人

#### 2.5.1 功能描述
- 回答用户关于加密货币的知识性问题
- 可结合看板内其他组件的数据（新闻、价格、公司动态）进行回答

#### 2.5.2 Agent 流程设计

```
┌──────────────────────────────────────────────────────────────────────────────┐
│                    聊天机器人 Agent 流程                                        │
└──────────────────────────────────────────────────────────────────────────────┘

[1] 用户输入接收
    │
    └─► 输入: 用户问题（文本 / 语音转文本）

                    ▼

[2] 意图识别 & 路由 Agent
    │
    ├─► 分类: 知识问答 | 价格查询 | 新闻查询 | 公司动态查询 | 闲聊
    └─► 输出: 意图类型 + 提取的关键实体（币种、公司名等）

                    ▼

[3] 知识检索 Agent（RAG）
    │
    ├─► 知识库: 加密货币百科、常见问题、看板内新闻/动态摘要
    ├─► 检索: 向量相似度检索 Top K 相关文档
    └─► 输出: 检索到的上下文片段

                    ▼

[4] DeepSeek 生成 Agent
    │
    ├─► 输入: 用户问题 + 检索上下文 + 系统 Prompt
    ├─► Prompt 设计: 角色设定、回答规范、引用来源
    └─► 输出: 自然语言回答

                    ▼

[5] 后处理 & 展示
    │
    ├─► 可选: 添加相关链接、价格卡片、新闻摘要
    └─► 输出: 格式化回复给用户
```

#### 2.5.3 数据流示意

```
用户问题 → 意图识别 → 知识检索(RAG) → DeepSeek生成 → 格式化回复
```

## 三、技术栈建议

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 前端 | React / Vue + ECharts / Recharts | 看板 UI + 趋势线图表 |

| 消息队列 | Redis / RabbitMQ | 异步任务（可选） |

## 六、安全与合规考虑

### 8.4 成本优化建议

1. **价格数据**：优先用 Binance/ByBit/Bitget 免费公开 API，避免 CoinGecko 付费
2. **新闻**：先用 RSS 验证需求，再考虑 Crypto News API
3. **Twitter**：X API 成本高，可评估第三方聚合（如 Nitter RSS）或降低采集频率
4. **DeepSeek**：利用缓存、精简 prompt、批量请求以降低 token 消耗
5. **站内公告**：自建爬虫成本最低，Apify 适合快速上线
